## 43. 람다보다는 메서드 참조를 사용하라

### 메서드 참조

- 람다 표현식이 단 하나의 메소드만을 호출하는 경우에 해당 람다 표현식에서 불필요한 매개변수를 제거하고 사용

```java
// 문법
{Class이름}::메소드이름
// or
{참조변수이름}::메소드이름

// 예제
(base, exponent) -> Math.pow(base, exponent)
// 메소드 참조
Math::pow
```

- 람다가 익명 클래스보다 나은 점은 간결함이다
- 함수 객체를 람다보다도 간단하게 만드는 방법이 있는데, 이를 메서드 참조

```java
// 자바 8에서 추가된 Map - merge() -> 람다를 활용해서 작성
// merge() 에서는 키, 값, 함수를 인수로 받음
// 키가 이미 있다면 (세 번째 인수로 받은) 함수를 현재 값과 주어진 값에 적용 후 현재 값에 덮어씀
map.merge(key, 1, (count, incr) -> count + incr);
```

```java
// 메서드의 참조를 전달하면 똑같은 결과
// 정적 메서드 sum 을 제공
map.merge(key, 1, Integer::sum)
```

- 메서드 참조에는 기능을 잘 드러내는 이름을 지어줄 수 있음
    - 친절한 설명을 문서로 개발 가능

```java
// 메서드 참조
servicee.execute(GoshThisClassNameIsHumongous::action)

// 람다 대체
service.execute(() -> action());
```

- 항상 메서드 참조가 간결하고 명확한 것은 아니다

### 메서드 참조 이점

- 매개변수 수가 늘어날수록 메서드 참조로 제거할 수 있는 코드 양 증가
- 람다로 구현했을 때, 너무 복잡하다면 메서드 참조 사용
- 기능을 잘 드러내는 이름을 지어줄 수도, 친절한 설명을 문서로 남길 수도
